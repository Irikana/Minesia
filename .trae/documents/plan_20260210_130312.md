# 问题分析

## 错误原因
- **错误类型**：参数越界错误 (`ArgumentOutOfBoundsError`)
- **错误信息**：`Value: 20, Argument bounds: [0, 10]`
- **错误原因**：单位不匹配
  - player.json 中设置的是"颗心"数量（"max": 50）
  - 我们的代码使用的是"半颗心"数量（20点 = 10颗心）
  - API 期望的是与 player.json 一致的单位

## 当前情况
- player.json 中：
  - "value": 10 （10颗心）
  - "max": 50 （50颗心）
- 我们的代码：
  - baseMaxHealth = 20 （20点半颗心 = 10颗心）
  - 传递给 API 的值 = 20
- API 错误：20 超出了 [0, 10] 的范围

## 解决方案

### 1. 修复 `updatePlayerAttributes` 函数
- 将生命值值从"半颗心"转换为"颗心"
- 确保值在 API 允许的范围内
- 与 player.json 中的设置保持一致

### 2. 实现步骤
1. **修改 `actions.js` 文件**：
   - 修改生命值计算逻辑，使用"颗心"作为单位
   - 确保传递给 API 的值正确
   - 保持属性加成的逻辑不变

2. **验证修复**：
   - 检查代码是否正确编译
   - 确保系统能够正常运行
   - 测试装备组合时的生命值计算
