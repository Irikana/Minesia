# 攻击速度冷却系统实现计划

## 1. 核心功能设计
- **全局攻击冷却系统**：为所有玩家添加攻击冷却机制
- **默认冷却时间**：设置为0.5秒（10 ticks）
- **物品特定冷却**：支持为不同工具和物品设置不同的冷却时间
- **攻击检测**：监听玩家攻击事件，触发冷却
- **冷却状态管理**：维护每个玩家的冷却状态

## 2. 实现步骤

### 步骤1：创建攻击冷却模块
- 在 `scripts/equipment/` 目录下创建 `attack_cooldown.js` 文件
- 实现攻击冷却状态管理
- 实现攻击事件监听器
- 实现冷却时间控制逻辑

### 步骤2：修改 main.js
- 导入攻击冷却模块
- 在主循环中集成冷却状态检查
- 确保冷却系统与现有装备系统兼容

### 步骤3：扩展装备规则系统
- 在 `rules.js` 中添加攻击冷却时间配置选项
- 支持为不同物品设置不同的冷却时间
- 集成到现有装备规则处理流程

### 步骤4：增强 actions.js
- 添加 `attack_cooldown` 动作类型
- 实现冷却时间应用逻辑
- 确保与现有状态管理系统集成

## 3. 技术实现细节

### 攻击冷却状态管理
- 使用 Map 存储每个玩家的冷却结束时间
- 每tick检查冷却状态
- 冷却期间阻止玩家攻击

### 物品特定冷却配置
- 在装备规则中添加 `attackCooldown` 属性
- 优先级：物品特定冷却 > 默认冷却

### 事件监听
- 使用 `world.afterEvents.entityHitEntity` 事件
- 检测玩家作为攻击者的情况
- 触发冷却计时

### 兼容性考虑
- 确保与现有装备效果系统无冲突
- 保持系统响应速度（1tick间隔）
- 错误处理和日志记录

## 4. 预期效果
- 玩家攻击后会进入冷却状态
- 默认冷却时间为0.5秒
- 可以通过装备规则为特定物品设置不同冷却时间
- 冷却系统全局应用，不影响其他游戏机制
