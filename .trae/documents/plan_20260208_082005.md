# 灵魂值HUD元素实现计划

## 功能需求
为灵魂系统添加一个HUD元素，具有以下特性：
- 行为类似生命值、饥饿值和盔甲值
- 展示在饥饿值上方
- 每10点灵魂值对应一个灵魂HUD元素
- 不与其他HUD元素矛盾

## 实现方案

### 1. 资源包修改

#### 1.1 添加灵魂值纹理
- **新建 RP/textures/gui/soul_bar.png**
  - 创建灵魂值条的纹理
  - 包含完整和空的灵魂值图标

#### 1.2 修改UI定义
- **新建 RP/ui/hud_screen.json**
  - 定义灵魂值HUD元素的位置和行为
  - 确保显示在饥饿值上方
  - 设置正确的布局和大小

#### 1.3 更新纹理定义
- **修改 RP/textures/terrain_texture.json**
  - 添加灵魂值纹理的引用

### 2. 行为包修改

#### 2.1 扩展灵魂系统
- **修改 BP/scripts/soulSystem.js**
  - 添加灵魂值变化事件
  - 实现与HUD元素的通信

#### 2.2 添加UI控制脚本
- **新建 BP/scripts/soulHud.js**
  - 处理灵魂值变化时的HUD更新
  - 确保HUD元素正确显示

### 3. 核心功能实现

#### 3.1 灵魂值计算
- 每10点灵魂值对应一个HUD元素
- 最大值100点对应10个完整的灵魂元素
- 部分值显示相应的填充程度

#### 3.2 HUD位置
- 显示在饥饿值上方
- 与其他HUD元素保持适当间距
- 确保在不同屏幕尺寸上正确显示

#### 3.3 状态更新
- 灵魂值变化时实时更新HUD
- 玩家加入时初始化HUD状态
- 确保HUD状态与实际灵魂值同步

### 4. 技术实现

- 使用Minecraft Bedrock Edition的UI系统
- 基于纹理的HUD元素
- 通过脚本控制HUD状态
- 与现有灵魂系统集成

## 预期结果
- 灵魂值HUD元素正确显示在饥饿值上方
- 每10点灵魂值对应一个HUD元素
- HUD元素随灵魂值变化实时更新
- 与其他HUD元素无冲突
- 系统稳定运行，无性能问题
