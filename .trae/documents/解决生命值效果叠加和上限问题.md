# 问题分析

## 当前实现
- 生命值增加通过 `health_boost` 药水效果实现
- 紫水晶碎片提供 `health_boost` amplifier 1（2点生命值）
- 盾牌套装提供 `health_boost` amplifier 0（1点生命值）
- 药水效果在Minecraft中**不会叠加**，只会取最高等级
- `health_boost` 效果每级只提供2点生命值，且有等级限制

## 存在的问题
1. **叠加问题**：同时装备多个提供同一属性的装备时，效果不会叠加
2. **上限问题**：无法通过药水效果提供大量属性加成
3. **通用性问题**：需要一个通用系统处理生命值、伤害、护甲等多种属性
4. **未来扩展性**：系统需要支持未来添加的新效果和属性

# 解决方案

## 1. 实现通用属性管理系统
- 创建新的属性计算逻辑，不再依赖药水效果
- 支持多种属性：生命值、伤害、护甲、速度等
- 累积计算所有装备和套装提供的属性加成
- 提供属性读取和修改的统一接口

## 2. 修改 `actions.js`
- 添加通用属性管理模块
- 为每种属性实现累积计算逻辑
- 定期更新玩家实际属性值
- 支持属性百分比修改功能

## 3. 扩展规则系统
- 允许在规则中直接指定属性加成数值
- 支持更灵活的属性配置
- 支持属性百分比修改动作

## 4. 实现步骤
1. **修改 `actions.js`**：
   - 添加通用属性管理逻辑
   - 实现属性加成累积计算
   - 添加读取和使用属性值的功能
   - 实现属性百分比修改

2. **更新 `rules.js`**：
   - 修改现有规则使用新的属性系统
   - 添加示例规则展示多种属性效果

3. **测试验证**：
   - 确保叠加效果正常工作
   - 测试多种属性的同时修改
   - 验证大量属性加成

4. **性能优化**：
   - 确保系统运行流畅
   - 优化属性更新频率

# 预期效果
- 装备多个提供同一属性的物品时，效果会正确叠加
- 可以轻松配置高达20点或更多的属性加成
- 系统支持生命值、伤害、护甲等多种属性
- 未来添加新效果时系统会自动适应
- 系统运行稳定，不影响其他游戏机制
