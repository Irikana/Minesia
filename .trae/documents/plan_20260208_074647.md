# 自定义灵魂组件实现计划

## 功能需求
创建一个代表实体灵魂的自定义组件，具有以下特性：
- 玩家初始载入世界或重生时默认值为100
- 可指定其他实体拥有该组件
- 灵魂值为0时，以2点每20tick的速度减少生命值

## 实现方案

### 1. 创建灵魂系统脚本
- **新建 BP/scripts/soulSystem.js**
  - 实现灵魂值管理逻辑
  - 处理玩家载入和重生事件
  - 实现灵魂值为0时的生命值减少逻辑

### 2. 扩展实体定义
- **修改 BP/entities/player.json**
  - 添加灵魂组件定义
  - 设置默认值为100

### 3. 集成到现有系统
- **修改 BP/scripts/main.js**
  - 导入并初始化灵魂系统
  - 确保与现有装备系统兼容

### 4. 核心功能实现

#### 灵魂系统核心功能
- **初始化功能**：玩家载入或重生时设置默认灵魂值100
- **值管理**：提供API用于修改灵魂值
- **状态检查**：定期检查灵魂值状态
- **生命值减少**：当灵魂值为0时，每20tick减少2点生命值

#### 技术实现
- 使用 `@minecraft/server` API
- 存储灵魂值到实体组件或使用Map存储
- 使用 `system.runInterval` 进行定期检查
- 处理实体载入和重生事件

### 5. 可扩展性
- 支持为其他实体添加灵魂组件
- 提供配置选项调整默认值和减少速率
- 与现有装备系统集成，允许装备影响灵魂值

## 预期结果
- 玩家初始拥有100点灵魂值
- 灵魂值为0时持续减少生命值
- 系统稳定运行，与现有功能兼容
- 提供API用于未来扩展
