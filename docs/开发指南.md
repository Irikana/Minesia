# Minesia 开发指南

## 名称来源

Minesia（又名创世），名字来源是取 Minecraft 的前 5 个字母与 Principia 的后 3 个字母构成，但考虑发音，后来变成了 Minecia，又因为可能引起歧义，把 c 换成了 s，最后变成 Minesia。

---

## 开发约定

开发 Minesia 时，应当注意一些约定俗成的规则，以免造成代码污染，以及管理混乱。

### 1. 新物品注册规则

当添加新物品时：

- **伤害范围**：如果物品具有伤害，应当是一个范围，请将该特性注册到 `scripts/random_damage` 下
- **体力消耗**：如果物品具有体力消耗，请将该特性注册到 `scripts/stamina` 下
- **自定义事件**：如果有自定义事件，记得注册到 `scripts/custom_events` 下
- **Lore 描述**：刚才所提到的这些内容都应当包含在 lore 描述当中

### 2. 功能模块管理

| 功能类型 | 管理目录 | 说明 |
|----------|----------|------|
| 随机伤害（伤害） | `scripts/random_damage` | 所有伤害相关逻辑 |
| 体力消耗 | `scripts/stamina` | 所有体力相关逻辑 |
| 套装效果 | `scripts/set_effect` | 包括一切对槽位具有要求的自定义事件的触发事件 |
| Lore 描述 | `scripts/lore_system` | 所有物品的 lore 描述管理 |

### 3. 价值映射关系

附加包里有默认约定好的价值映射关系：

#### 货币价值

| 货币 | 等价关系 |
|------|----------|
| 1 个绿宝石币 | ≈ 8 个钻石币 |
| 1 个钻石币 | ≈ 9 个金币 |
| 1 个金币 | ≈ 8 个银币 |
| 1 个银币 | ≈ 11 个石币 |
| 1 个石币 | ≈ 14 个木币 |

> ⚠️ **注意**：红宝石币作为神秘的非法货币，不具备用其他货币衡量价值的能力，且万万不可在未特意说明时，为它创造任何可能的获取途径。

#### 物品价值

| 物品 | 等价关系 |
|------|----------|
| 1 个腐肉 | ≈ 1 个木币 |
| 1 个铁锭 | ≈ 4 个石币 |
| 1 个金锭 | ≈ 5 个银币 |
| 1 个钻石 | ≈ 3 个金币 |
| 1 个下界合金锭 | ≈ 4 个钻石币 |

> 📝 **说明**：价值映射关系仅仅代表着一种配置参考，而非固定的关系，附加包中的内容在合理范围内允许价值波动以增加可玩性。

### 4. 目录说明

| 目录 | 说明 |
|------|------|
| `builds/` | 仅用于 bridge 快速构建，本质上不会影响任何开发和调试进程，在开发创世时应当忽略 |

### 5. 武器类型特性

新增武器时，如果声明了镰刀特性（在物品名称或描述或自定义事件中），在加入到附加包时应当按具体情况选择自定义事件中的镰刀横扫事件。大多数时候加入一把镰刀武器，都默认其能够触发镰刀横扫事件，此为武器类型特性，如果没有特意声明不需要添加至 lore。
